useContext+reducers+local storage

介绍

在React中，Context API提供了一种简洁又干净的方法在不同组件中共享状态。 其典型应用场景包括： 用户认证，界面语言，主题等“全局”性的设置。


Source: Pixabay

使用Context Provider

要想使用上下文，首先需要创建上下文的提供者： Context Provider。 比如：想要创建一个和用户认证相关的上下文。

首先，添加src/contexts/LoginContext.js:

import React, { createContext, useState } from 'react';

export const LoginContext = createContext();

export default function LoginContextProvider({ children }) {

    const [isAuthenticated, setIsAuthenticated] =  useState(false);
    const [role, setRole] =  useState('public');

    const login = () => {
        setIsAuthenticated(true);
    }

    const logout = () => {
        setIsAuthenticated(false);
        setRole('');
    }

    return (
        <LoginContext.Provider value={{ isAuthenticated, login, logout, role, setRole }}>
            { children }
        </LoginContext.Provider>
    )
}

对其进行调用的组件：

import React, { useContext } from 'react';
import {LoginContext} from '../contexts/LoginContext';

export default function LoginTest() {
    const loginContext = useContext(LoginContext);
    const { isAuthenticated, login, logout, role, setRole } = loginContext;

    return (
        <div style={{margin: "25px"}}>
            You are {isAuthenticated? "logged in.": "logged out."}<br/>
            Your role: {role}<br/>
            <button onClick={isAuthenticated? logout:login}>{isAuthenticated? "Logout":"Login"}</button>
            <button onClick={() => setRole('Admin')}>Change role to Admin</button>
            <button onClick={() => setRole('public')}>Change role to public</button>
        </div>
    )
}

在App.js中，

<LoginContextProvider>
    <LoginTest/>
</LoginContextProvider>

使用Context Provider保存全局数据

有些时候甚至可以把数据放到Context Provider中，比如：

import React, { createContext, useState } from 'react';

export const BookContext = createContext();

const BookContextProvider = ({children}) => {
  const [books, setBooks] = useState([
    {title: 'Java程序设计', id: 1101},
    {title: 'Python程序设计', id: 1102},
    {title: 'React in Action', id: 1103},
  ]);
  return (
    <BookContext.Provider value={{books}}>
      {children}
    </BookContext.Provider>
  )
}

export default BookContextProvider;

在组件中使用BookContext:

const { books } = useContext(BookContext);

这样就可以在不同组件中共享 books 实例。

使用reducer

首先，在使用Context Provider的时候，不是必须使用reducer的，但如果使用，可以简化代码。比如，在CustomerContextProvider中，我们可以添加addCustomer, removeCustomer等方法并把它们传递给使用Context的组件中，但这样做对于简单应用还好。对于稍微复杂的应用，就会过于烦琐了。最好的方法是使用reducer将这些方法统一进行管理。

创建reducer

首先，需要创建一个Reducer，并将操纵状态的逻辑放在reducer中：

import uuid from 'react-uuid';

export const customerReducer = (state, action) => {
  switch (action.type) {
    case 'ADD_CUSTOMER':
      return [...state, {
        name: action.customer.name, 
        phone: action.customer.phone, 
        id: uuid()}
      ];
    case 'REMOVE_CUSTOMER':
      return state.filter(customer => customer.id !== action.id);
    case 'ADD_DUMMY_CUSTOMERS':
      return [
        {id: uuid(), name: 'Stuart', phone: '07853110066'},
        {id: uuid(), name: 'Lucas', phone: '07853110067'},
        {id: uuid(), name: 'Andy', phone: '07853110068'}
      ];    
    case 'RESET_CUSTOMER':
      return [];    
    default:
      return state;
  }
}

在ContextProvider中使用reducer

接下来，在CustomerContext中，就不要使用useState，而是使用useReducer了。因为state的管理已经转移到reducer中了。

import React, { createContext, useReducer } from 'react';
import { customerReducer } from '../reducers/customerReducer';

export const CustomerContext = createContext();

const CustomerContextProvider = (props) => {
  const [customers, dispatch] = useReducer(customerReducer, []);
  return (
    <CustomerContext.Provider value={{ customers, dispatch }}>
      {props.children}
    </CustomerContext.Provider>
  );
}

export default CustomerContextProvider;

在UI组件中使用Context和reducer

接下来在UI组件中只需要通过传递给dispatch相应的参数就可以调用相应的状态更新方法了：

import React, { useContext } from 'react';
import { CustomerContext } from '../contexts/CustomerContext';

export default function CustomerList() {

    const customerContext = useContext(CustomerContext);
    const { customers, dispatch } = customerContext;

    return (
        <div>
            Current customers (click to delete):
            <hr/>
            <ul>
            {customers.map(customer => {
                return ( 
                    <li key={customer.id} onClick={() => dispatch({ type: 'REMOVE_CUSTOMER', id: customer.id })}>{customer.name} / {customer.phone}</li>
                );
            })}
            </ul>
            <button onClick={() => dispatch({ type: 'ADD_DUMMY_CUSTOMERS' })}>Add dummy customers</button>
            <button onClick={() => dispatch({ type: 'RESET_CUSTOMER' })}>Reset customers</button>
        </div>
    )
}

Local Storage

key value pairs

值必须为字符串类型。

需要将JSON转换为字符串后才能够保存到local storage。
customer = {“name”:”George”,”phone”:”07882223333”}
localStorage.setItem(‘c1’, JSON.stringify(customer))

因此在读取后需要再次转换为JSON：
george = JSON.parse(localStorage.getItem(‘c1’))

如果想要使用local storage，在CustomerContext.js中：

const [customers, dispatch] = useReducer(customerReducer, [], () => {
    const localData = localStorage.getItem('customers');
    return localData ? JSON.parse(localData) : [];
});

useEffect(() => {
    localStorage.setItem('customers', JSON.stringify(customers));
}, [customers]);
