React Hooks学习笔记

自从React 16.8推出了Hooks之后，很多React开发者都开始由类式组件转向了函数式组件，尤其是在新的项目中，大量应用函数式组件。通过Hooks，可以实现类式组件中的状态，组件生命周期(只支持三种回调：componentDidMount, componentDidupdate, componentWillUnmount)，甚至还能实现全局上下文，以及类似于Redux的全局状态管理。可以说，一般情况下，函数式组件能够满足所有的需求。但在一些特殊场合，尤其时特别复杂的组件中，仍然需要类式组件。


Source: Pixabay

useState

通过数组的解构赋值获得状态，及更新状态的方法。由于是数组的解构赋值，因此只要按照索引顺序写即可，变量名可以随意（当然，在真实项目中不能随意。）。useState的唯一参数是状态的初始值。

import React,{useState} from 'react';
…...
const [count,setCount] = React.useState(0);

如果需要使用多个状态，则多次调用useState:

const [title,setTitle] = useState('');
const [content , setContent] = useState('');

函数式组件本身就相当于类式组件的render方法，因此每当重新渲染时都会被调用。需要注意：传递给setState的初始值只会被赋值一次，这是因为React底层对此做了相应处理，在第二次之后的调用中不会再次用初始值覆盖当前状态值。

在调用setCount的时候，一般情况下都是直接把新的状态值传递给React就可以了。但在处理逻辑复杂的时候也可以传递一个函数给setXXX：

setCount((count)=>{自定义处理逻辑并返回新的状态值});

useEffect

好，回到正题，在默认方式下不传递第二个参数)，useEffect就相当于类式组件中的componentDidMount及 componentDidUpdate方法，在组件挂载及任何状态更新后会被调用：

React.useEffect(() => {
    console.log(“useEffect的默认使用方式”);
});

因此，其默认使用方式是不适合调用初始化操作的。如果想要调用一些初始化操作，比如发出网络请求获取数据，启动定时器，设置消息订阅等，应该在使用时添加第二个参数：一个空数组。这时候，useEffect就只相当于类式组件中的componentDidMount了。这时，空数组代表不监测任何状态的更新。

React.useEffect(() => {
    console.log(“实现相应的初始化操作”);
}, []);

也可以在数组中提供想要监测的状态，这时候就会在组件挂载，以及被监测状态变化时被调用：

React.useEffect(() => {
    console.log(“实现相应的初始化操作”);
}, [状态1，状态2]);

如何在函数式组件中实现类式组件中的componentWillUnmount回调函数，比如在组件卸载之前关闭定时器？useEffect函数的第一个参数的返回值就相当于componentWillUnmount。

// 以下时组件的卸载代码
ReactDOM.unmountComponentAtNode(document.getElementById('root'))

// 通过useEffect调用定时器/网络请求等
React.useEffect(()=>{
    let timer = setInterval(()=>{
        //自己的定时器 
    }, 1000)
    // 在useEffect中，其返回值相当于componentWIllUnmount的回调函数
    return () => {
        clearInterval(timer)
    }
}, [])

useRef

userRef可以在函数组件中存储/查找组件内的数据，它在函数式组件中实现不同输入控件交互时非常有用。

首先需要注册一个Ref，下面的userName就是相应的名字：

const userName = React.useRef()

接下来，在添加文本框的时候，需要指明将要把文本框和前面指定的ref容器绑定：

<input type="text" ref={userName}/>

当需要在其他组件中访问这个文本框的值时，需要使用userName.current.value：

const show = () => {
  console.log(userName.current.value);
}

<button onClick={show}>Output username</button>

useContext

useContext可用于在父子组件之间传递消息。

首先，需要注册context:

const myContext = React.createContext();

接下来需要设定Provider，也就是上下文的提供者，这里是父组件。正如下面所示，通过给myContext.Provider传递一个对象属性，来实现在子组件中访问/更新父组件状态。这里传递给value的值是一个对象，包括父组件状态名和更新父组件状态的方法(dispatch)。

export default function UseContextDemo() {
    // 通过useState hook设定初始状态值
    const [name, setName] = React.useState("Julie");

    return (
        <myContext.Provider value={{name, setName}}>
            <div>
                <Child/>
            </div>
        </myContext.Provider>
    )
}

在子组件中，可以通过React.useContext(myContext)来获取父组件传递过来的状态及更新状态的方法(dispatch)：

const Child = props => {
    const {name, setName} = React.useContext(myContext);
    return(
        <div>
            Current name: {name}<br/>
            <button onClick={() => setName("Jack")}>Change name</button>
        </div>
    )
}

需要注意的是，这种上下文的传递不仅仅局限于直接的父子组件，孙子辈的组件照样能够获取爷爷组件传递过来的属性/操作属性的方法(dispatch)：

import React from 'react'

const myContext = React.createContext();

const GrandChild = props => {
    const {name, setName} = React.useContext(myContext);
    return(
        <div>
            In GrandChild, current name: {name}<br/>
            <button onClick={() => setName("Lucas")}>Change name</button>
        </div>
    )
}

const Child = props => {
    const {name, setName} = React.useContext(myContext);
    return(
        <div>
            Current name: {name}<br/>
            <button onClick={() => setName("Jack")}>Change name</button>
            <GrandChild/>
        </div>
    )
}

export default function UseContextDemo() {
    const [name, setName] = React.useState("Julie");

    return (
        <myContext.Provider value={{name, setName}}>
            <div>
                <Child/>
            </div>
        </myContext.Provider>
    )
}

useReducer

useReducer是一个用于状态管理的Hooks API，它接受两个函数，分别为reducer函数和初始状态。

useReducer(reducer, initialState)

看一个完整的例子：

import React from 'react'

export default function UseReducerDemo() {
    const initState = 0;
    const reducer = (state, action) => {
        switch(action) {
            case 'add':
                return state + 1;
            case 'minus':
                return state - 1;
            case 'reset':
                return initState;
            default:
                return state    
        }
    }

    const [count, updater] = React.useReducer(reducer, initState);

    return (
        <div>
            <h2>Count: {count}</h2>
            <button onClick={()=>updater('add')}>Add</button>
            <button onClick={()=>updater('minus')}>Minus</button>
            <button onClick={()=>updater('reset')}>Reset</button>
        </div>
    )
}

上面的例子中state和action分别都是简单类型：整数/字符串，我们可以把它们都变为对象，这样能处理的逻辑就可以更复杂了，比如：对于action:

switch (action.type) {
    case 'add':
      return {
        count: state.count + action.value
      }

对于其调用部分：

<button onClick={() => dispatch({type: 'add', value: 5})}>Add</button>

在开发的时候，经常会用到多个reducer：

const [count, setCount] = React.useReducer(countReducer, 0);
const [name, setName] = React.useReducer(nameReducer, 'Nobody');

将useContext和useReducer结合

正如前面useContext部分介绍的一样，可以通过useContext在父子组件间传递状态，无论多少级都可以。那么，也可以把处理复杂业务的reducer通过useContext传递给子组件，这样，就可以在子组件中通过调用从父组件传递过来的dispatch更新父组件状态。

可以实现如下的效果。

其中，A是B和C的父组件。D是C的子组件。

代码如下：

import React from 'react'

const countContext = React.createContext();

export function A() {
    const {count, updater} = React.useContext(countContext);
    return (
        <div>
            In component A: {count}<br/>
            <button onClick={()=>updater('add')}>Add</button>
            <hr/>
            <B/>
            <C/>
        </div>
    )
}

export function B() {
    const {count, updater} = React.useContext(countContext);
    return (
        <div>
            In component B: {count}<br/>
            <button onClick={()=>updater('add')}>Add</button>
        </div>
    )
}

export function C() {
    const {count, updater} = React.useContext(countContext);
    return (
        <div>
            In component C: {count}<br/>
            <button onClick={()=>updater('add')}>Add</button>
            <hr/>
            <D/>
        </div>
    )
}

export function D() {
    const {count, updater} = React.useContext(countContext);
    return (
        <div>
            In component D: {count}<br/>
            <button onClick={()=>updater('add')}>Add</button>
        </div>
    )
}


export default function UseContextUseReducerDemo() {
    const initState = 0;
    const reducer = (state, action) => {
        switch(action) {
            case 'add':
                return state + 1;
            case 'minus':
                return state - 1;
            case 'reset':
                return initState;
            default:
                return state    
        }
    }

    const [count, updater] = React.useReducer(reducer, initState);

    return (
        <countContext.Provider value={{count, updater}}>
            <div>
                <h2>Count: {count}</h2>
                <button onClick={()=>updater('add')}>Add</button>
                <button onClick={()=>updater('minus')}>Minus</button>
                <button onClick={()=>updater('reset')}>Reset</button>
                <A/>
            </div>
        </countContext.Provider>
    )
}

useState和useContext比较容易搞混。可以把握以下原则：

  • 如果在一个组件内，应使用useState
  • 如果要实现复杂的业务逻辑时，使用useReducer
  • 如果state内保存的是基本数据类型，比如整型，字符串的话，使用useState
  • 如果在多个组件间传递状态时，使用useReducer
  • 如果状态经常变化，建议统一使用useReducer，便于统一管理
